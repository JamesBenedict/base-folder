IDS Build-Out Template

This is a general template to publish long form articles at the Indiana Daily Student. If you have any questions, feel free to come talk to me at the IDS Mondays and Wednesday from 3 - 4 p.m. or email me at jabenedi@indiana.edu.

Developed by James Benedict

This template is adapted from Waiting for Help published in the Spring of 2015 at the Indiana Daily Student. It is being developed for stories from Bonnie Layton's multimedia section of the Words and Pictures course at IU in the Fall of 2015.

Read and follow all comments in the html if you wish to publish at the IDS. Call in ids.css and ids.js last in your page unless you are using another plugin that requires its file to be loaded last. If something isn't changing after you style it in the style.css, it might be that the ids.css is over riding it. If this is the case, you can ask to alter they style, but you need a justification.

Media Categories

There are two categories your media falls into. Some are contained in figure tags, other are html content you will make custom divs to hold. In general, html content scales size easier, but figure content is easier to setup. All figure media

  • embedded content

  • graphics
  • stylized text
  • maps
  • timelines
  • soundcite
  • storymaps
  • anything else you request

Media Placement

The grid of this article on a medium or larger screen is one body-copy column that is 7 columns wide with one column offset to the left, and one sidebar column that is 4 columns wide. For mobile, everything is the full 12 colunmns wide. These columns live in a row which can be used to organize sections of your code. You can copy in as many columns into a row as you want, but once the total width of the columns equals 12, any other columns will float underneath. Since everything floats to the top, the easiest way to lineup the sidebar is the include the sidebar column directly after your body-copy column. Then the sidebar will float and line up with the top of the first paragraph or media in the body-copy column. If you want to read more about the grid structure, read about Bootstrap scaffolding.

Any media can be placed in one of three areas - inline, sidebar on break. Inline is in the body copy column, sidebar is obviously in the sidebar and break breaks from the row and gives you access to make something the full screen width. You can technically add more options by making divs in the columns a different percentage width (width: 50%, 150%, etc.) if you need a work around. Try to avoid doing this if you can, but often when making graphics something doesn't line up quite right and you need a little more freedom. Stay away from absolute values (width: 50px) as they are more buggy when coding a responsive design unless something is small (like styling the twitter button to width: 5px).
